# BizCard 产品定位与功能总览

> **视角**：高级产品官梳理  
> **状态**：当前共识版本（面向 Kickstarter 及后续软件演进）  
> **更新**：2026-01

---

## 一、产品定位（一句话）

**BizCard 是一款个人向的效率工具硬件 + App**：以录音与 AI 处理为核心，覆盖「会议当下」与「闪念速记」两种输入场景，通过 E-ink 屏幕提供实时反馈与呈现，并通过 App 完成转写、清理、提醒与日历的闭环。

---

## 二、核心价值主张

| 维度 | 表述 |
|------|------|
| **主叙事** | 会议时：**不用看手机也能看清要点与翻译**；随时：**按一下就记闪念，录完即得贴心反馈**。 |
| **与竞品差异** | 竞品（Plaud / 钉钉 / TicNote）以「贴手机背的录音卡」为主、无屏；BizCard **有 E-ink 屏**，可做实时展示与情感化反馈，并可延伸**桌立形态**（屏朝用户），从「藏在背后」升级为「放在面前」。 |
| **软件定位** | 硬件的最佳搭档：录音 → 转写/清理 → Reminder & Calendar；Contacts / Profile / 名片 / AI Twin 为配套能力，不抢硬件主叙事，在完整用户旅程与长期留存中再强化。 |
| **电子名片的定位** | **电子名片是 E-ink 与软件能力的延展，不是主叙事**：主叙事是效率（会议 + 闪念）；「碰一碰发名片」则是在「有屏 + 有 Profile/URL」之上的延伸场景——NFC 负责快速传递链接，E-ink 负责展示「名片面」，软件提供可配置的 URL（名片页即一个 URL）。对外可提，但不作为卖硬件的核心故事。 |

## 三、硬件能力与形态

### 3.1 硬件核心能力

- **录音**：会议场景（长时、环境音）+ 闪念场景（用户主动触发、≤1 分钟）。
- **E-ink 屏幕**：  
  - **会议场景**：在实现「蓝牙/手机实时 AI 转录」后，作为**实时总结与翻译的呈现载体**（用户可桌立使用，屏朝自己）。  
  - **闪念场景**：录完给出**正反馈 emoji**，由 AI 根据输入内容的情绪自动选择，增强「被理解」与互动感。
  - **电子名片（延展）**：碰一碰分享时，E-ink 可展示「名片面」（如姓名、Slogan），与 NFC 传递的 URL 配套。
- **NFC 芯片**：支持**分享（Card 模拟）**；若硬件支持 **Reader 模式**，还可**读外部 NFC 标签**，实现「获取」与「触发」。详见下节。
- **闪念 Trigger**：按钮或触摸，用户主动触发「当前这段是闪念」，与会议录音在交互与后处理上区分。

### 3.2 NFC 的三种用法（分享 / 获取 / 触发）

| 用法 | 方向 | 能力说明 | 前提 |
|------|------|----------|------|
| **分享** | 对方手机读卡片 | 卡片作为被动标签，传出在 App 中配置的链接（如「我的名片」URL）。碰一碰对方手机即传链接。电子名片 = NFC 传递 + E-ink 展示名片面 + 软件配置 URL。 | 芯片支持 Card 模拟（已有） |
| **获取** | 卡片读外部 NFC 标签 | 用户用卡片去碰场所/物体上的 NFC 标签，卡片**读取**标签内数据（短文本、URL 或 ID）。数据可上传 App 形成「某时某地」记录，或在 E-ink 上展示（如 "Checked in @ Room B"）。 | 芯片支持 **Reader 模式** |
| **触发** | 同上，读标签驱动动作 | 读到的内容（或「读到」本身）在固件/App 中定义为**触发器**：如切换模式、带场所/场景开始录音、在 E-ink 显示清单、签到打卡等。 | 芯片支持 **Reader 模式** |

若仅有 Card 模拟，则仅能「分享」；支持 Reader 后即可做场所签到、碰标签打上下文录音、离场检查等延展场景。

### 3.3 形态差异化（建议）

- **可贴 + 可立**：支持贴手机背面携带，同时支持**桌立**（折叠/支架/卡槽），开会时设备放桌面、屏朝用户，便于瞥看实时总结/翻译，而不必看手机。
- **双场景统一**：同一块屏在会议中「展示信息」，在闪念后「展示情绪反馈」，形成一致的产品心智。

## 四、E-ink + NFC + 录音 延展场景与创意功能（可选路线图）

在「会议 + 闪念」主叙事之外，三者组合可支撑的延展能力（部分依赖 NFC Reader 模式）：

| 场景类型 | 典型功能 | 用到的能力 |
|----------|----------|------------|
| **分享后的反馈** | 对方存完名片后，你的 E-ink 显示「Just saved」或感谢图标 | E-ink + NFC（被读事件） |
| **场所签到 / 打标签** | 碰现场/会议室 NFC 标签 → 本条或后续录音带「场所」信息；E-ink 显示 "Checked in @ X" | NFC 获取/触发 + E-ink |
| **碰标签再录** | 碰「Room B」或「今日一句」标签 → 开始带上下文的录音；E-ink 显示当前场景 | NFC 触发 + 录音 + E-ink |
| **换名片后闪念** | 碰对方手机发完名片后，自动进入短闪念（如 15–30s）记一句，录完 emoji，闪念自动挂到该联系人 | NFC（分享完成事件）+ 录音 + E-ink |
| **会后一句牢记** | 会议结束，AI 抽一句「最该记住的事」在 E-ink 保留一段时间，回工位瞥一眼即知 | E-ink + 会议管线 |
| **待机展示** | 空闲时 E-ink 显示今日焦点、昨日闪念摘要或自定义一句 | E-ink + 软件 |
| **离场 / 仪式** | 碰「下班前检查」标签 → E-ink 显示清单；或碰「今日一句」→ 短录音 + emoji，数据进日记/闪念流 | NFC 触发 + E-ink + 录音 |

上述为创意池与优先级候选，不做主叙事；实现节奏依硬件能力（是否支持 NFC Reader）与资源排期而定。

## 五、双模输入与 AI 处理策略

| 模式 | 触发 | 时长 | AI 处理 | E-ink 输出 |
|------|------|------|---------|------------|
| **会议** | 会议开始/环境 | 长（数分钟～数十分钟） | 转写 + 总结/要点，可选实时翻译 | 实时总结/翻译（未来） |
| **闪念** | 用户按/摸硬件 | 短（≤1 分钟） | 仅：去语气词 + 去冗余 + 一句 paraphrase；**不做**会议式长总结与全文转录 | 录完后根据**情绪**由 AI 选一个正反馈 emoji；情绪判断**不依赖转录**，可用音频特征或短文本快判 |

## 六、软件能力分层（当前认知）

### 6.1 与 Kickstarter 强相关的（可轻量提及）

- **录音同步与存储**：会议 + 闪念统一进入 App。
- **会议管线**：转写 → 总结 → 提炼 Reminder / Action → 进入 Calendar。
- **闪念管线**：轻清理（去语气词 + paraphrase）→ 以「语音备忘录/闪念」形式展示，可选加入 Reminder。

### 6.2 配套能力（有则呈现，不强行写进主叙事）

- **Profile**：个人数字名片，可分享、可被扫描。
- **Contacts**：联系人管理；来源包括分享、交换、扫码添加纸质名片。
- **Share / Exchange / Scan**：发自己名片、换他人名片、扫纸质名片入库。
- **AI Digital Twin**：基于 Profile 的线上挡驾、答疑、接线 bot；可归纳为「你不在时接盘、线索留痕」，在完整叙事中作为第四段（始终在线），Kickstarter 阶段一句带过即可。

### 6.3 叙事节奏建议

- **众筹阶段**：主叙事 = 硬件 + 会议实时呈现 + 闪念与 emoji 反馈；Contacts/Profile/名片/AI Twin 以「App 还支持……」轻量带过。
- **后续**：再展开「身份 → 建联 → 当下 → 事后 → AI Twin」的完整故事，用于留存、订阅与 B2B。

## 七、体验与情感定位

- **闪念 + 情绪 emoji**：产品体验上可以带有「会回应、懂一点你」的**灵性**，对外表述为「根据你刚说的内容给出贴心的正反馈」或「懂你一点的小助理」，**不主动定性为「电子宠物」**，避免与专业/效率主叙事冲突；若用户自发说「像小宠物」可作 UGC 传播。

## 九、一句话总结（对外/对内复用）

> **BizCard：个人效率工具——会议时用 E-ink 看实时总结与翻译，随时按一下就记闪念并得到懂你的 emoji 反馈；App 把录音与闪念变成可执行的提醒与日程，联系人、名片与 AI 分身则把身份与关系接在一起。**
