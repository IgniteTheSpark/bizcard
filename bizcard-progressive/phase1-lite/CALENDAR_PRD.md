# BizCard Calendar & Reminder PRD
# 日历与提醒 - 产品需求文档

> **版本**：v1.0  
> **目标**：以轻量日历模式打通"录音-会议-提醒-人"的闭环  
> **设计理念**：一个 Calendar，统一管理 Meeting 和 Reminder

---

## 1. 背景与目标

### 1.1 核心问题

当前 BizCard 应用需要解决的关键问题：**让用户进入 App 后知道应该做什么**。

需要解决的问题：
1. Meeting 记录与后续行动没有关联
2. Contact 详情的 Activity 只有标题，缺乏上下文
3. 录音产生的价值停留在"记录"，没有转化为"提醒"

### 1.2 产品目标

> **核心理念**：From "Recording" to "Reminding"（从记录到提醒）

**目标**：
- ✅ 用户进入首页即可看到今日会议和近期提醒事项
- ✅ 会议记录能自动/手动产生 Reminders
- ✅ 提供完整的 Calendar 页面管理日程（Meeting + Reminder）
- ✅ Contact 页面能展示与该联系人相关的完整上下文
- ✅ 形成"录音 → 会议 → Reminders → 完成"的闭环

### 1.3 首页设计理念

**首页内容排序逻辑**：
1. **Profile Card**：快速分享和预览入口
2. **Today's Meeting**：**核心内容**，对应用户"硬件录音 → 回App查看"的主要动作
3. **Reminders Hub**：轻量日历提醒，按时间分组

**设计要点**：
- 首页中心是 **Meeting** 和 **Reminder**
- Meeting 在 Reminders Hub 之前，因为遵循用户"硬件录音→回App查看"的主要动作流程
- Reminders Hub 采用轻量日历模式，按 Today / Next 7 Days 分组

### 3.2 Today's Meeting

**位置**：在 Reminders Hub 之前

**功能**：
- 显示今天的所有会议

#### 4.3.1 周日期条

**功能**：
- 显示当前周的 7 天日期
- 点击日期切换选中日期
- 左右滑动切换周

**双色圆点指示**：
- 🔵 蓝色圆点 = 有 Meeting
- 🟠 橙色圆点 = 有 Reminder
- 两者都有时显示两个圆点

**日期样式**：
- 今天日期：蓝色圆形背景
- 选中日期：浅蓝色背景
- 其他月份日期：灰色文字

#### 4.3.2 筛选 Tab

**位置**：选中日期标题下方

**选项**：

| Tab | 显示内容 |
|-----|----------|
| All | Meeting + Reminder 混合列表（默认） |
| Meetings | 只显示 Meeting |
| Reminders | 只显示 Reminder |

#### 4.3.3 混合时间流

**显示逻辑**：
- 选中某个日期后，下方显示该日期的所有 Meeting 和 Reminder
- 按时间排序：Meeting 有具体时间，Reminder 没有时间则放在后面

**排序规则**（同一天内）：
1. Meeting 按时间排序（早到晚）
2. Reminder 放在最后

**Tab 切换效果**：
- **All**：显示 Meeting + Reminder 混合列表，有分组标题
- **Meetings**：只显示 Meeting，无分组标题
- **Reminders**：只显示 Reminder，无分组标题

#### 4.3.4 日历视图交互

| 操作 | 行为 |
|------|------|
| 点击某日期 | 选中该日期，下方显示该日内容 |
| 左右滑动周日期条 | 切换到上一周/下一周 |
| 点击 `1月 ▼` | 打开 Date Picker |
| 点击 Meeting 卡片 | 进入 Meeting 详情页 |
| 左滑 Meeting 卡片 | 显示删除按钮 |
| 点击 Reminder 勾选框 | 标记完成 |
| 点击 Reminder 卡片其他区域 | 打开 Reminder 详情页（Bottom Sheet） |
| 左滑 Reminder 卡片 | 显示删除按钮 |

> **注意**：卡片内不再有内联编辑交互（日期、联系人、标题点击），所有编辑在 Reminder 详情页完成。详见第7章。

#### 4.4.1 布局说明

**筛选 Tab**：
- 位于列表顶部，sticky 固定
- 与日历视图共享筛选状态

**日期分组**：
- 按日期倒序显示（最近的在上面）
- 相对日期显示：今天、昨天、明天
- 其他日期显示：`X月X日 周X`
- 空日期自动跳过（不显示没有数据的日期）

**浮动 Today 按钮**：
- 当今天的日期不在可视区域时显示
- 点击快速滚动到今天

#### 4.4.2 无限滚动

**初始加载**：
- 从今天开始
- 往前加载 14 天
- 往后加载 7 天

**向下滚动**（加载更早的历史）：
- 触发条件：滚动到距离底部 < 300px
- 加载内容：再加载前 14 天的数据
- 显示 Loading 指示器

**向上滚动**（加载未来日程）：
- 触发条件：滚动到顶部
- 加载内容：再加载后 7 天的数据

### 5.2 AI Suggested Reminders

当 Meeting 处理完成后，AI 自动从 Transcript 中提取待办事项。

**提取规则**：
- 识别含有动作意图的句子（如"我会..."、"下周要..."、"需要跟进..."）
- 识别明确的任务分配（如"Kevin 负责..."）
- 识别约定的事项（如"周五前完成..."）

**展示和交互规范**：

```
┌────────────────────────────────────────┐
│  ✨ AI Suggested                       │
│  ─────────────────────────────────────  │
│  ┌──────────────────────────────────┐  │
│  │ Follow up with Kevin on RWA      │  │
│  │          [✗ Dismiss] [✓ Accept]  │  │
│  └──────────────────────────────────┘  │
│  ┌──────────────────────────────────┐  │
│  │ Send updated proposal by Friday  │  │
│  │          [✗ Dismiss] [✓ Accept]  │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

**关键交互规则**：

| 规则 | 说明 |
|------|------|
| **AI Suggested 不可编辑** | 用户不能直接修改 AI 建议的文案 |
| **两个操作按钮** | [✓ Accept]（接受）和 [✗ Dismiss]（拒绝） |
| **Accept 后移入 My Calendar** | 点击 Accept 后，该项移到 My Calendar 区域 |
| **Dismiss 后消失** | 点击 Dismiss 后，该建议消失 |
| **My Calendar 中可编辑** | 进入 My Calendar 后，用户可以修改文案、设置日期等 |

**点击 [✓ Accept] 后的流程**：
1. 该建议项从 AI Suggested 区域消失（带动效）
2. 同时出现在下方的 My Calendar 区域
3. 显示 Toast："Added to My Calendar ✓"
4. 如果所有建议都已处理，显示 "✨ All AI suggestions processed"

**点击 [✗ Dismiss] 后的流程**：
1. 该建议项从 AI Suggested 区域消失（带淡出动效）
2. 不会添加到 My Calendar
3. 可选：显示 Toast："Dismissed"

## 8. 全局交互规范

### 8.1 完成 Reminder 反馈

当用户勾选完成一个 Reminder：
1. 勾选框显示打勾动画
2. 标题添加删除线效果
3. 底部显示 Toast："Marked as done ✓"
4. 计数器更新

### 8.2 删除 Reminder

**方式一：左滑删除**（快捷操作）
1. 左滑 Reminder 卡片，显示删除按钮
2. 点击 🗑 Delete 按钮
3. 显示确认对话框
4. 确认后删除
5. 显示 Toast："Reminder deleted"

**方式二：详情页删除**
1. 点击 Reminder 卡片进入详情页
2. 滚动到底部点击"🗑 Delete Reminder"
3. 显示确认对话框
4. 确认后删除
5. 关闭详情页
6. 显示 Toast："Reminder deleted"

### 8.2.5 删除 Meeting

**左滑删除**（快捷操作）
1. 左滑 Meeting 卡片，显示删除按钮
2. 点击 🗑 Delete 按钮
3. 显示确认对话框
4. 确认后删除 Meeting 及其关联的所有 Reminders
5. 显示 Toast："Meeting deleted"

> **注意**：删除 Meeting 会同时删除所有关联的 Reminders，确认对话框中应提示用户。

### 8.3 修改日期

在 Reminder 详情页：
1. 点击快捷按钮（Today/Tomorrow）或日历图标
2. 选择新日期
3. 日期显示更新
4. 点击 Save 保存

| 下拉 Popover | 关闭 Popover |
| 点击遮罩层 | 关闭 Popover |

| **Already Added** | 已关联的联系人 | 显示勾选状态 ✓，可取消 |
| **All Contacts** | 所有可选联系人 | 包含 Self 选项，按字母排序 |

**交互逻辑**：

| 操作 | 行为 |
|------|------|
| 点击已添加联系人 | 取消勾选，从关联列表移除 |
| 点击未添加联系人 | 勾选，添加到关联列表 |
| 点击 Self | 勾选/取消勾选 Self |

**Self 选项说明**：
- Self 选项**始终出现在 All Contacts 列表顶部**
- 无论当前是否已有其他联系人，用户都可以选择添加 Self
- Self 和其他联系人可以**同时存在**（例如：Self + Kevin + Alice）

**边界情况**：

| 情况 | 处理 |
|------|------|
| 取消所有联系人 | 允许，显示 "+ Add contact" |
| 只选 Self | 允许，显示 "👤 Self" |
| Self + 其他人 | 允许，显示 "👤 Self, Kevin +1" |
| 搜索 "self" | Self 选项会被匹配到 |

### 8.6 Self 联系人处理

"Self" 表示用户自己，用于个人提醒或无明确联系人的情况。

**产生场景**：用户独自录音、AI 未识别到联系人、用户手动创建个人提醒。

**显示规则**：

| 场景 | 卡片显示 |
|------|----------|
| 只有 Self | 👤 Self |
| Self + 其他人 | 👤 Self, Kevin +1 |
| 无联系人 | No contacts |

**交互规则**：
- Self 在联系人选择器中**始终可选**，位于列表顶部
- Self 可以和其他联系人**同时存在**
- 点击 Self 联系人不会跳转（因为是自己）
- 所有编辑在 Reminder 详情页完成
