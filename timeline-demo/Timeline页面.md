# BizCard Activity Timeline

## 意义

Timeline 通过时间轴将线下的 **"接触锚点"**（录音/会面/名片交换）与线上的 **"AI 接洽"**（点击/Agent 对话）串联起来。它不仅还原了互动全过程，更通过 **Action Center** 将社交碎片转化为待办任务，填平了从"认识"到"成交"之间的执行鸿沟。

---

## 数据类型

| **数据类型** | **展示位置** | **处理逻辑** |
| --- | --- | --- |
| **Agent Conversation** | **Timeline (大卡片)** | 核心业务。展示访客意图摘要。 |
| **Offline Meeting** | **Timeline (大卡片)** | 核心资产。展示地点 (Pin) 与总结。 |
| **Contact Exchange** | **Timeline (大卡片)** | 展示交换地点 (Pin) 与对方身份摘要。 |
| **General Visit** | **Timeline (圆点)** | 轻量信号。蓝色点 🔵。 |
| **Click Action** | **Timeline (圆点)** | 深度交互。紫色点 🟣。 |
| **Contact Request** | **Timeline (圆点)** | 日志记录。黄色点 🟡。 |

## 组件类型

### A. 大卡片 (Hero Cards)

_交互：点击进入对应详情页；卡片仅保留任务提示徽章。_

| **卡片类型** | **头部 (Header)** | **内容 (Body Content)** | **提示徽章 (Badge)** | **点击行为** |
| --- | --- | --- | --- | --- |
| **Meeting Card** | 时间 | **对象:** 姓名<br>**地点:** 📍 地点名称<br>**摘要:** 一句话会面总结 | `[🔴 N Action Items]` (仅在有未处理任务时显示) | 进入会面详情页 |
| **Exchange Card** | 时间 | **对象:** 头像 + 姓名 + 职位<br>**地点:** 📍 交换地点<br>**公司:** 所属企业名称 | `[🔴 N Action Items]` (如有后续跟进需求) | 进入联系人名片页 |
| **Agent Card** | 时间 | **访客:** Visitor #ID（匿名）或 姓名（已注册用户）<br>**意图:** AI 摘要的对话内容 | `[🔴 1 Action Item]` (待回复意图) | 进入 Agent Call 详情页 |

> **说明：** Visitor #ID 表示非 BizCard 注册用户，即使在对话中透露身份，只要未注册即为 Visitor。

### B. 小圆点 (Micro Dots)

_交互：默认堆叠展示，点击展开。展开后纯日志展示，无进一步交互。_

| **颜色** | **含义** | **动作规范 (Action)** |
| --- | --- | --- |
| 🔵 **Blue** | **General Visit** | "Viewed Profile" |
| 🟣 **Purple** | **Click Event** | "Clicked \[链接/文件名\]" |
| 🟡 **Yellow** | **Contact Request** | "Sent Contact Request" |

### 原型说明

#### 时间轴逻辑 

*   时间锚点 
    
    *   左侧垂直时间轴**不是**每分每秒都显示时间，只有以下两种情况显示具体时间文本：
        
        *   **Big Cards (大卡片):** 每一个大卡片左侧必须显示发生时间 (例如 "9:20 AM")。
            
        *   **Whole Hours (整点):** 跨越整点时显示刻度 (例如 "10:00 AM")，作为时间路标。
            
*   智能虚化
    
    *   如果是普通的小圆点 (Dot) 或聚合点 (Cluster)，左侧时间轴**留白**，不显示时间。
        
    *   如果某时间段之前无数据，轴线头部渐隐，不显示空白刻度。
        
#### 空状态设计（空状态 + 历史衔接）

当用户今天没有任何互动记录时，采用「空状态 + 历史衔接」的展示方式：

1. **始终从今天开始** → 用户知道当前日期
2. **今天无数据** → 显示简洁的内联空状态卡片（引导分享名片）
3. **自动衔接历史** → 空状态下方直接显示最近有数据的日期内容
4. **日期标签优化** → 显示 "(2 days ago)" 让用户快速感知时间距离

```plaintext
+---------------------------------------------------+
|                                                   |
|  JAN 08, 2026 (Today)                             |
|                                                   |
|  +---------------------------------------------+  |
|  |  📭 No activities today.                    |  |
|  |     Share your BizCard to get started!      |  |
|  |                    [ 📤 Share BizCard ]     |  |
|  +---------------------------------------------+  |
|                                                   |
|- - - - - - - - - - - - - - - - - - - - - - - - - -|
|                                                   |
|  JAN 06, 2026 (2 days ago)                        | <--- 最近有数据的日期
|                                                   |
| 2:30 PM                                           |
|   📍   ┌──────────────────────────────────────┐   |
|   │    │ MEETING            [ 🔴 1 Action ]   │   |
|   │    │ Investor Update Call                 │   |
|   │    │ 📍 Zoom (Online)                     │   |
|   │    │ 👤 Sarah Kim                         │   |
|        └──────────────────────────────────────┘   |
|                                                   |
+---------------------------------------------------+
```

#### 特殊情况处理

| 场景 | 展示方式 |
| --- | --- |
| **今天有数据** | 正常展示今天内容 |
| **今天无数据，昨天有** | 内联空状态 + 昨天内容 |
| **今天无数据，最近是3天前** | 内联空状态 + "JAN 05 (3 days ago)" |
| **完全没有任何数据（新用户）** | 完整空状态引导页（分享名片 + 查看 Actions） |

#### 滚动与加载逻辑

*   **默认定位：** 优先展示今天的 Timeline。

*   **预加载策略：** 默认加载今天 + 最近1-2天有数据的内容。

*   **懒加载：** 继续向下滚动时，按天为单位加载更早的数据。

*   **日期分隔符：** 无数据的日期不显示分隔符，直接跳到有数据的日期。

#### 交互逻辑

*   堆叠
    
*   **场景：** 11:05 发生会面，11:06 发生 Agent 对话。
    
*   **逻辑：** 忽略时间轴的物理比例，优先保证卡片内容完整展示。
    
*   **表现：** 两张大卡片上下**紧贴排列**，垂直撑长线条，绝不重叠遮挡。

*   **排序规则：** 同一天内按时间戳倒序排列（最新的在最上方）。

### 原型说明

#### Date Picker 交互逻辑

为了保持操作的流畅性，采用 **"点选即始，再点即止"** 的逻辑，无需单独切换"单日/范围"模式。

**选择器规则**

*   **入口：** 点击 Timeline 右上角的 `[📅]` 图标。
    
*   **弹窗：** 底部半屏弹窗 (Bottom Sheet)。
    
*   **点击逻辑：**
    
    1.  **第一次点击 (例如 Nov 19):**
        
        *   系统判定为 `Start Date`。
            
        *   视觉上该日期高亮（实心圆）。
            
        *   底部按钮文案变为："查看 Nov 19 的动态"。
            
    2.  **第二次点击 (例如 Nov 21):**
        
        *   系统判定为 `End Date`。
            
        *   视觉上 Nov 19 到 Nov 21 之间的区域打通（浅色背景连接）。
            
        *   底部按钮文案变为："查看 Nov 19 - Nov 21 的动态"。
            
    3.  **特殊情况 (再次点击 Start Date):**
        
        *   如果用户点了 Nov 19，又点了 Nov 19。
            
        *   确认为"单日选择"。
            

*   **限制 (Constraints):**
    
    *   **未来日期：** Disable (置灰不可点)。
        
    *   **6个月前：** Disable (与数据策略保持一致，6个月后数据归档只读)。

#### 不同 Tab 的行为差异
        
| Tab | 点击日期后的行为 |
| --- | --- |
| **Activities** | Timeline 跳转到该日期位置 |
| **Actions** | 筛选该日期的 Action Items |

#### Header 展示逻辑

当用户处于"筛选状态"时，Header 需要承担 **"告知范围"** 和 **"退出筛选"** 两个功能。

*   **默认状态 (浏览流):** 显示 `TODAY` 或 `Timeline`。
    
*   **筛选状态 (Filter Mode):** Header 变为高亮或特定样式，明确显示时间段。
    
| **场景** | **Header 文案格式** | **右侧操作** |

| **单日选择** | `Nov 19, 2024` | `[X]` (清除筛选) |

| **跨年范围** | `Dec 28, 2024 - Jan 2, 2025` | `[X]` (清除筛选) |

*   **交互动作：** 点击 `[X]`，Header 恢复为 `TODAY`，列表重置为全量最新数据。

#### 关键细节

1.  **日期分隔符 (Date Divider):**
    
    *   它不能打断轴线（Axis），轴线应该**穿过**日期文字，或者日期文字作为轴线的一个"结"。
        
    *   建议样式：日期居中，两侧有虚线或细实线连接到主轴。
        
2.  **整点锚点 (Whole Hours):**
    
    *   在每一个日期区块内部，依然遵循"整点显示"的规则。
        
    *   **重置规则：** 每天开始时，不需要重复显示上一个时段的整点，直接从该天最新的时间点开始。
        
3.  **空数据处理：**
    
    *   如果 `Nov 19` 有数据，`Nov 20` 没数据，`Nov 21` 有数据。
        
    *   选择 `19-21` 范围时，Timeline 里**直接不显示** `Nov 20` 的分隔符。
        
    *   视觉上就是：`Nov 21 内容` -> `Nov 19 内容`。这样最紧凑。

## Action Items Tab

该页面是用户的**待办清单**，支持日期筛选，每个条目直接关联社交上下文。

### 原型图例

```plaintext
+---------------------------------------------------+
| [ < ]           ACTION CENTER                [📅] | <--- 打开 Date Picker
+---------------------------------------------------+
| [ 17 ]  [ 18 ]  [ 19🔴 ]  [ 20 ]  [ 21🔴3 ]  [ 22 ] | <--- 横向快速导航
+---------------------------------------------------+
|                                                   |
|  +-----------------------------------------------+|
|  | FROM: Meeting with Kevin Du (Jan 08)     [→] ||<--- 点击跳转详情页
|  |-----------------------------------------------|
|  | [ ] Send follow-up email                      ||
|  | [ ] Send API documentation                    ||
|  | [ ] Schedule demo next week                   ||
|  |                          [✨ Follow-up]       ||<--- Group 级别的 CTA
|  +-----------------------------------------------+|
|                                                   |
|  +-----------------------------------------------+|
|  | FROM: Agent Call #402 (Jan 08)           [→] ||
|  |-----------------------------------------------|
|  | [ ] Reply to pricing inquiry                  ||
|  |                          [✨ Follow-up]       ||
|  +-----------------------------------------------+|
|                                                   |
+---------------------------------------------------+
```

### 原型说明

#### 横向日历栏

*   **作用：** 快速导航，一目了然哪些天有待办任务。

*   **展示：** 显示近期约 7 天，左右滑动可延展。

*   **红点数字：** 显示该天未完成的 Action Items 数量。

*   **交互：** 点击日期筛选该天的 Action Items。

#### Action Items 分组展示

*   **分组逻辑：** 同一个 Meeting/Agent Call 的 Action Items 归为一组。

*   **来源卡片：** 显示来源类型、对象名称、日期。

*   **跳转入口 [→]：** 点击跳转到对应的 Meeting/Agent Call 详情页。

*   **[✨ Follow-up] CTA：** 针对该组生成统一的跟进消息。

#### 复选框交互

*   用户手动勾选 mark as done。

*   完成后：复选框变为 ✓，文字加删除线 + 变灰。

*   已完成的任务保留在列表底部或收起到「已完成」折叠区。

## 搜索功能

### 入口

Timeline 右上角的 `[🔍]` 图标。

### 搜索范围

关键词搜索，匹配以下字段：
- 联系人姓名
- 公司名称
- 会面摘要内容
- Agent 对话摘要

### 搜索结果展示

搜索结果以 Timeline 形式展示，按时间倒序排列匹配的节点。

## 删除功能

### 支持场景

用户可删除任意 Timeline 记录（Meeting、Agent Call、Exchange 等）。

### 交互逻辑

1. 长按或进入详情页后点击删除按钮
2. 弹出二次确认弹窗
3. 确认后：
   - 该记录从 Timeline 移除
   - 关联的 Action Items 一并删除
