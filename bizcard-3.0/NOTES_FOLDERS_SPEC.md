# Notes 文件夹与多选功能说明

参考 TicNotes、Plaud 等竞品，在 Notes 首页补充「文件夹」与「多选处理」能力，便于用户对 Note 进行归类与批量操作。本文档约定功能点与两种首页入口方案。

---

## 1. 文件夹：创建、编辑、删除

### 1.1 创建文件夹

- **入口**：在「文件夹」选择/筛选区域提供「+ 新建文件夹」或「+」按钮（具体入口见第 3 节）。
- **创建弹窗内容**（参考竞品「创建新文件夹」）：
  - **名称**：必填；占位符可写「必填」；为空时「创建」按钮置灰。
  - **颜色**：可选；提供若干预设色块（如 6–8 种）供选择，用于文件夹在列表中的标识。
  - **图标**：可选；点击进入图标选择页，选中的图标作为文件夹在列表中的展示图标。
- **操作**：底部「创建」按钮；关闭按钮（如右上 X）。创建成功后关闭弹窗并刷新当前文件夹/列表。

### 1.2 编辑文件夹

- **入口**：在某个文件夹项右侧提供「更多」操作（如 `...`），点击后弹出菜单，含「编辑」。
- **编辑**：可修改名称、颜色、图标（与创建时字段一致）；保存后刷新展示。

### 1.3 删除文件夹

- **入口**：同上「更多」菜单，提供「删除」；建议用红色或危险样式区分。
- **逻辑**：删除前可二次确认（如「确定删除该文件夹？其中的 Note 将移回 My Notes」）。删除后，原属于该文件夹的 Note 归入默认「My Notes」，不随文件夹物理删除。

### 1.4 默认视图与层级（当前方案）

- **全部文件**：固定选项，用于默认呈现用户所有文件；不是文件夹，不可编辑/删除。
- **My Notes**：与用户自建文件夹（如「工作」）**同一层级**，仅为用户首次进入 app 时系统默认创建的一个文件夹；新 Note 默认归属此处；用户可编辑、可删除，删除后其内 Note 需有归属策略（如归入其他文件夹或新建默认文件夹，产品可定）。
- **用户文件夹**：用户创建的文件夹，与 My Notes 同级展示，每项带数量与「…」编辑/删除。
- **暂不提供**：未分类视图、回收站兜底逻辑；后续若需回收站再单独设计。

## 2. 多选与批量操作

### 2.1 进入多选模式

- **方式**：对某条 Note **左滑**后出现操作项，其中一项为「多选」或「选择」；点击后进入多选模式。
- **备选**：部分竞品在列表顶部提供「选择」入口，或长按某条进入多选；可后续与左滑方案二选一或并存。
- **多选模式表现**：
  - 列表每条左侧/右侧出现**勾选框**；已选条目标记为选中态。
  - 顶部标题区显示「已选中 (n)」及「完成」按钮（退出多选）。
  - 底部出现**批量操作栏**，仅在多选模式下展示。

### 2.2 批量操作项（参考竞品）

- **移动到**：选择目标文件夹，将当前选中的 Note 移入该文件夹（若已在则不变）。
- **重命名**：单条时直接进入重命名；多条时可为多条统一加前缀/后缀或逐条处理（按产品取舍）。
- **合并**：将选中的多条 Note 合并为一条（如合并 transcript/摘要，需定义合并规则）。
- **删除**：将选中 Note 移至回收站或软删除；建议二次确认。
- **生成**（可选）：将选中 Note 作为输入，生成 Idea（即创建一条类型为「想法」的 Note，关联来源 Note；见 Notes 架构中的 IDEA 类型）。

具体首期上线哪些操作，可按实现成本与优先级裁剪。

### 2.3 退出多选

- 点击顶部「完成」或等价入口，清空选择并恢复为普通列表浏览。

## 3. 首页「文件夹 / 筛选」入口的两种方案

在首页 Notes 区域，需要有一个入口让用户**选择当前要看的文件夹或视图**（全部、未分类、某文件夹、回收站），并可选**排序**（如按修改时间）。竞品有两种常见形态：

### 3.1 方案 A：从下弹起的近全屏 sheet + 单按钮类型筛选（当前实现，参考 Plaud）

- **位置**：标题行为「当前视图 ▼」（如 全部文件 (5)）；**类型筛选**为单按钮「筛选」+ 下拉（全部/会议/闪念/想法）；与「+」并列。
- **视图/文件夹入口**：点击「当前视图 ▼」后**从底部弹起一个近全屏的 sheet**（约 88% 高度），而非小下拉。标题为「筛选和排序」，带拖拽条与关闭按钮（X）。
- **Sheet 内容**：排序（修改时间）、固定选项「全部文件」、文件夹区块（My Notes 与用户文件夹同级，每项「…」编辑/删除）。**「…」弹出的编辑/删除菜单始终在 sheet 内**，避免被裁切或错位。
- **优点**：大容器内点「…」时选项弹窗也在同一窗口内，体验稳定；类型单按钮不占空间。
- **缺点**：文件夹很多时 sheet 内列表需滚动。

### 3.2 方案 B：侧边栏筛选与排序（TicNotes 式）

- **位置**：在首页主区域提供入口，如标题栏左侧的「筛选/菜单」图标，或从左侧边缘右滑。
- **交互**：打开一个**侧边栏/抽屉**，标题为「筛选和排序」，内容包含：
  - **排序**：如「修改时间」+ 升序/降序。
  - **视图/筛选**：全部文件、未分类、回收站、以及「文件夹」区块；文件夹区块带「+」新建，每个文件夹右侧「…」可编辑/删除。
  - 可选「来自」等按来源/类型的筛选（如按 Meeting / Memo）。
- **优点**：空间大，适合文件夹多、筛选维度多的场景；新建/编辑/删除文件夹都在同一侧边栏内完成。
- **缺点**：需要常驻或明显的侧边栏入口，占一定布局。

### 3.3 与现有 Notes 筛选的关系

- 当前已有 **全部 / 会议 / 闪念 / 想法** 的类型筛选（Ideas 归类为 Notes 的一种类型）。
- **文件夹**是「归属」维度（放在哪个文件夹），**全部/会议/闪念/想法** 是「类型」维度；二者可并存：
  - 先选「当前视图」：全部 / 未分类 / 某文件夹 / 回收站；
  - 再在列表上方或侧边栏内保留类型筛选：全部 / 会议 / 闪念 / 想法。
- 排序（如修改时间）可放在下拉或侧边栏顶部，对当前视图下的列表生效。

## 4. 与结构化资产的关系

- **Note** 在数据模型上建议增加可选字段：`folderId`（或 `folderIds` 若支持多文件夹）。未归类时为空，对应「未分类」视图。
- **文件夹**为应用层组织概念，可不作为独立「资产类型」持久化到与 Note/Event 同级的模型，仅作为 Note 的归类维度；若需要同步、分享文件夹，再考虑单独建模。
- 删除文件夹不删除 Note，仅将其中 Note 的 `folderId` 置空；回收站逻辑与 Note 的软删除策略一致，见产品回收站规范。

## 5. 验收要点小结

| 能力 | 要点 |
|------|------|
| 文件夹创建 | 弹窗含名称（必填）、颜色、图标；创建按钮在名称为空时禁用。 |
| 文件夹编辑/删除 | 文件夹项有「更多」→ 编辑 / 删除；删除前确认，删除后 Note 归未分类。 |
| 多选入口 | 左滑某条 Note 可进入多选模式；列表出现勾选与「已选中 (n)」、底部操作栏。 |
| 批量操作 | 至少支持「移动到（文件夹）」「删除」；可选重命名、合并、生成。 |
| 首页入口 | 方案 A：标题处下拉「筛选和排序」+ 文件夹列表；方案 B：侧边栏「筛选和排序」+ 文件夹列表。 |

方案 A / B 可在一个 PRD 中二选一，或不同首页方案（如 Option1 / Option2）采用不同入口形态以做 A/B 验证。
