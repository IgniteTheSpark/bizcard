# 03 意图路由器与指令历史 (Intent Router & History)

## 设计背景
加入 E-ink 硬件后，用户通过“长按语音输入”与系统交互的频率会大幅提升。但并非所有的长按都是为了“记录一段内容”（例如灵感闪念），有很大一部分其实是向 AI 下达“指令”（例如“帮我找一下冯总的联系方式”）。

如果将这两类截然不同的语音全当成 Note 存起来，会导致信息流极为混乱，且无法给用户提供即时的结果反馈。

## 核心实现：Intent Router (意图分发器)
我们在 ASR (语音转文本) 完成之后，增加了一层轻量级的**意图识别层**，称为 `IntentRouter`。

### 工作流
1. 接收到硬件传来的 ASR 逐字稿。
2. 内部通过大模型提示词（或轻量级启发式规则）判断用户意图：
   - **路径 A：内容沉淀 (MEMO Intent)**
     - **判断依据**：如“我想到一个好idea...”，“明天需要带上文件”。
     - **处理逻辑**：落库生成一条 `VOICE_MEMO` 类型的 Note，触发大模型进行长文本摘要总结，并提取专属待办事项 (Reminders)。
   - **路径 B：行为指令 (COMMAND Intent)**
     - **判断依据**：包含明确的行动指令，如“帮我查询”、“起草邮件”、“调用”。
     - **处理逻辑**：不再生成 Note，而是将指令透传给后端的 Ask Agent 进行处理（如调用 `bizcard-search` 等 Tools）。

## 语音指令历史沉淀 (Command History)
对于走**路径 B (行为指令)** 的交互，由于没有落为 Note，如果仅仅是在 E-ink 屏幕上闪现一次结果，会导致用户无法追溯。

因此，我们在 App 端新建了专门的**“语音指令交互页面”**。

- **展现形式**：类似对话流（Chat History）的时间线。
- **内容包含**：
  - 用户的原始语音识别文本及时间戳（来源标注为 From E-ink Device）。
  - Biz Assistant 返回的结果卡片（例如：解析出的联系人名片片段、草拟的邮件正文快捷查看链接）。
- **价值**：确保通过硬件发起的轻量查询不仅能实时反馈，还能在 App 端留下永久可查的数字资产。
