# BizCard 3.0 首页方案1 PRD（Notes First）

## 1. 方案定义

- 首页主区域仅展示 `Notes`。
- `Calendar` 不占首页主区域，作为 Top Bar 入口进入独立页面/独立视图。
- 目标：最大化首页“记录与回看”的主路径效率，降低首屏认知复杂度。

## 2. 适用场景

- 现阶段以硬件用户为主，用户预期是“进入即记录/查看记录”。
- Contacts、Calendar 均为次级能力，不应与 Notes 争夺首屏注意力。
- 重点优化新用户理解与高频录音用户的单路径效率。

## 3. 信息架构

1. Top Bar：
   - 个人中心（头像+名称）
   - 硬件状态（Connected/Disconnected）
   - Calendar 入口（进入独立 Calendar 页面）
2. **Top Bar 下方（主滚动区内、Notes 区块之上）**：
   - **Ask Agent 入口条**：全宽胶囊形态（如左侧 AI 图标 + 文案「Ask me anything…」）。**非首页内联输入**；用户**点击后进入 Ask Agent 专页**（与 `bizcard-askagent` 中「独立对话页、侧边 history」一致）。底部 Tab 若仍有居中 Ask，与此入口**同一套体验、同一目标页**。
   - **（预留，v1 可占位或隐藏）主动推送 Panel**：位于 **Ask 入口条与 Notes 标题行之间**，**可折叠**；后续承载今日简报、会前摘要、EOD 等 Agent 主动内容。折叠态控制高度；展开态与 Ask 闭环为「卡片/追问 → 进入 Ask 专页并带上下文」。详见 `AGENT_PROACTIVE_BRIEFING_DESIGN.md`。
3. 主区域（Notes）：
   - **Notes 标题与筛选**：首屏「当前视图 ▼」+ **「筛选」单按钮**（点击下拉：全部 / 会议 / 闪念 / 想法）+「+」；**下拉内仅排序 + 固定选项「全部文件」+ 文件夹（My Notes 与用户文件夹同级，均可编辑/删除）**。Ideas 归类为 Notes 的一种类型。详见 `NOTES_FOLDERS_SPEC.md`。
   - Notes 内容分组：`这周 / 更早`
   - **文件夹**：支持创建、编辑、删除文件夹；创建时可填名称（必填）、颜色、图标。
   - **多选模式**：左滑某条 Note 可进入多选；多选后支持批量「移动到文件夹」「重命名」「合并」「删除」等。详见 `NOTES_FOLDERS_SPEC.md`。
4. 底部悬浮高频动作（**仅两项**）：
   - **`Add note`**：主录音入口（大胶囊 + 麦克风图标）。
   - **`+`**：**上传录音文件**入口（圆形次要按钮）；点击后进入**上传本地音频**流程（与未连接硬件时「上传音频」能力一致）。
   - **不再**在底部单独放置「Ask me anything」按钮；Ask 以 **Top Bar 下入口条** + **底部 Tab 居中 Ask**（若保留）为主。

## 4. 关键交互

- **Ask Agent 入口条**：点击整条（或主热区）→ **导航至 Ask Agent 专页**（如 `ask-agent-demo.html` 或产品内路由）；不在首页拉起键盘或内嵌对话。
- **（预留）主动推送 Panel**：折叠/展开交互与内容优先级见 `AGENT_PROACTIVE_BRIEFING_DESIGN.md`；v1 可无内容或仅占位。
- **Notes 标题行「+」**（与筛选、当前视图并列）：点击弹出选项（与现有一致）：
  - `Add Reminder`
  - `Upload Audio`（与底部悬浮「+」互为补充入口，行为一致即可）
- **底部悬浮「+」**：**直接**触发「上传录音文件」流程（文件选择 → 进入与产品约定的导入/转写链路）；**不要求**先弹出二选一菜单（快捷上传）。
- 底部 **Add note** 点击采用“一体式展开过渡”后跳转：
  - **Add note**：若硬件已连接 → 动效后跳转 `capture-demo.html`；若**未连接硬件** → 不跳转，弹出「硬件连接提醒」弹窗。
- **未连接硬件时点击 Add note**（硬件连接提醒弹窗）：
  - 弹窗标题：请先连接硬件；说明文案：连接录音硬件后可开始实时采集；您也可以选择上传本地音频文件。
  - 操作选项：
    - **前往连接硬件**：关闭弹窗并跳转至硬件连接页（如 `hardware-connection.html` 或产品约定的连接页）。
    - **上传音频**：关闭弹窗并进入**上传本地音频**流程（等同点击底部悬浮「+」或标题行「+」中的 Upload Audio）。
    - **取消**：关闭弹窗。
  - 点击遮罩可关闭弹窗；弹窗内点击不关闭。
- **文件夹与筛选**：类型筛选用单个「筛选」按钮，下拉为全部/会议/闪念/想法。点击「当前视图 ▼」展开下拉：排序 + 固定选项「全部文件」+ 文件夹（My Notes 与用户文件夹同级，均可编辑/删除）。Note 默认归属 My Notes；Ideas 为 Note 类型之一。暂不提供未分类、回收站。
- **多选与批量操作**：左滑某条 Note 进入多选模式，列表出现勾选与底部操作栏；支持批量移动到文件夹、重命名、合并、删除等（具体见 `NOTES_FOLDERS_SPEC.md`）。
- Calendar 入口进入独立页面，默认月历视图：
  - 默认定位当前月份与当前日期；
  - 右上支持 `Calendar / Timeline` 模式切换；
  - 内容仅展示 `Reminders`；
  - `Done` 为折叠区，支持展开查看全部已完成；
  - Timeline 模式默认按日期分组，展示从未来到过去的所有未完成 reminders。

## 5. 视觉与交互原则

- 一级路径单一：首屏主列表仍为 Notes，不混入 Calendar 内容；**Ask 以导航条进入专页**，不占用首屏对话流。
- 纵向层级：**Top Bar → Ask 条 →（可折叠主动区）→ Notes 标题与筛选 → 列表**；主标题（Notes）与筛选仍高于卡片元信息。
- **底部悬浮仅双控件**：录音主按钮 + 上传次按钮，避免第三枚并列 CTA 分散注意力。
- 科技感+简洁：冷色科技基底、轻玻璃容器、统一图标风格。
- 触控可用性：主要按钮最小点击区域 >= 44px。

## 6. 验收标准

- 首页主区域不出现 Calendar 内容。
- Top Bar 可直接进入 Calendar。
- **Top Bar 下方展示 Ask Agent 入口条**；点击后进入 **Ask Agent 专页**（非首页内联输入）。
- **Ask 条与 Notes 标题之间**具备接入**可折叠主动推送 Panel** 的结构位置（v1 允许无内容或占位）。
- **文件夹**：可创建（名称必填 + 颜色/图标可选）、编辑、删除文件夹；删除后其内 Note 归 My Notes。
- **筛选**：类型为单按钮「筛选」+ 下拉（全部/会议/闪念/想法）；视图下拉含排序、全部文件、文件夹（My Notes 与用户文件夹同级，「…」与文件夹同行居中对齐）（见 `NOTES_FOLDERS_SPEC.md`）。
- **多选**：左滑某条 Note 可进入多选模式；支持批量移动到文件夹、删除等。
- 类型与视图筛选生效后，分组（这周/更早）正确显示/隐藏。
- **Notes 标题行「+」** 弹层含 `Add Reminder` 与 `Upload Audio`（与 `NOTES_FOLDERS_SPEC` 一致）。
- **底部悬浮仅 `Add note` 与 `+`**：`+` 可直接触发上传录音文件；**底部无独立「Ask me anything」按钮**。
- **Add note** 可触发展开过渡并跳转采集页（或硬件未连接时出弹窗）。
- **硬件未连接时**：点击 Add note 不跳转，弹出硬件连接提醒弹窗；弹窗含「前往连接硬件」「上传音频」「取消」；上传音频进入与底部「+」一致的上传流程。
- **硬件已连接时**：点击 Add note 正常进入采集页。
- Calendar 独立页默认进入月历视图且仅展示 reminders。
- Calendar 独立页支持 `Calendar / Timeline` 切换与 `Done` 折叠查看。
