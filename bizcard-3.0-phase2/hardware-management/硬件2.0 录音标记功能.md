# 硬件2.0 录音标记功能

#### 1. 触发与物理反馈

*   **前置条件：** 设备正处于“会议”或“通话”录音状态中。
    
*   **用户动作：** **单击（或短按）**右上角角键。
    
    *   _补充：闪念触发阈值已统一为长按 **0.5s**（见《_硬件2.0 需求内容》与硬件管理 Proposal 的已拍板决策）。_
        
*   **震动反馈：** **连续两次极其短促的震动)**。
    
    *   _逻辑区分：_ 开录是长震，闪念是单次短震，打点是两次短震。用户不用看屏幕，靠手感就知道“标签打成功了”。
        

#### 2. 屏幕 UI 反馈

在按下的那 1~2 秒内，我们需要在 0.96 寸的屏幕上给一个视觉确认，然后迅速恢复到波形界面。

*   **Line 1 (系统)：** 保持不变（蓝牙 + 电量）。
    
*   **Line 2 (模式)：** 保持不变（显示时长或模式）。
    
*   **Line 3 (动态波纹区临时替换)：**  原本的波纹 `|||ııı||ıı|||` 瞬间被替换为一个 **图文组合提示**，例如：`🚩 MARKED` 或 `📌 重点`。
    
    *   停留 1.5 秒后，自动切回跳动的波纹。
        

---

### 🔄 状态机更新对照 (逻辑闭环)

现在，右上角这个按键（角键）的逻辑变得非常饱满且聪明（Context-Aware）：

| **当前设备状态** | **用户动作：操作右上角“角键”** | **系统执行结果** |
| --- | --- | --- |
| **待机中 / 未录音** | 长按 0.5s | ⚡ 触发 **闪念模式** (新建一个独立的极短录音文件)。 |
| **会议 / 通话录音中** | 单击 / 短按 | 🚩 触发 **打重点模式** (在当前的录音文件中插入一个时间戳锚点)，伴随双震动与屏幕临时提示。 |
