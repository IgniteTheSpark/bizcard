# 硬件2.0 蓝牙耳机连接流程

### 连接流程

#### Phase 1: 激活模式与连接手机音频链路 (音频源打通)

*   **步骤 1：功能入口触发**
    
    *   用户在 App 的"设备控制台"或"录音准备"页面，点击开启 **\[耳机录音模式\]** 的开关（或专属大按钮）。
        
*   **步骤 2：状态检测与跳出引导**
    
    *   App 后台迅速检测手机是否已连接 `Eureka-Audio`（经典蓝牙）。
        
    *   **如果没有连接：** App 底部弹出一个强阻断的半屏弹窗 (Bottom Sheet)。
        
    *   **弹窗 UI：**
        
        *   标题："需要连接手机音频设备"
            
        *   动画/图文引导："请前往系统蓝牙列表，找到并连接名为 `Eureka-Audio` 的设备。"
            
        *   操作按钮：一个醒目的 **\[前往系统蓝牙设置\]** 按钮（点击触发 Deep Link 直接跳入手机系统设置）。
            
*   **步骤 3：用户跳出与返回**
    
    *   用户在系统设置中点击 `Eureka-Audio` 完成连接。
        
    *   用户切回 Eureka App。App 监听到音频蓝牙已连接，弹窗自动收起，并在页面顶部给出一个短暂的绿色提示条 (Toast)："手机音频已成功路由"。
        

#### Phase 2: 耳机配对与接管 (桥接打通)

*   **步骤 4：耳机就绪引导**
    
    *   紧接着上一步，App 界面进入"寻找耳机"状态。
        
    *   **页面 UI：**
        
        *   核心文案："请将您的蓝牙耳机放入充电仓并长按按键，使其进入**等待配对状态**。"（可折叠显示 AirPods/华为等主流耳机的操作图示）。
            
        *   前置提醒："请确保耳机已与当前手机断开连接。"
            
        *   操作按钮：**\[耳机已就绪，开始扫描\]**。
            
*   **步骤 5：扫描与列表呈现**
    
    *   用户点击"开始扫描"。App 下发指令让卡片开启主蓝牙扫描。
        
    *   页面显示雷达波纹的 Loading 动画。
        
    *   App 动态渲染出一个 **\[附近可连接设备\]** 列表（过滤掉非音频设备，展示扫描到的耳机名称）。
        
*   **步骤 6：选择与最终确认**
    
    *   用户在列表中点击自己的耳机名称（例如 "Justin's AirPods Pro"）。
        
    *   列表上方显示 Loading 状态："正在建立中继连接..."。
        
    *   **连接成功：** App 页面状态大更新，显示 "耳机已成功接入"。
        
    *   _(此时，卡片硬件端只在这一刻配合展示一个连接成功的常驻小图标或状态指示即可)_。
        

### 卡片的 3 次蓝牙连接拆解：

#### 第 1 次连接：手机 App ↔ 卡片【控制链路】

*   **连接类型：** BLE（低功耗蓝牙 / Bluetooth Low Energy）
    
*   **发生时机：** 用户刚打开 Eureka App 并唤醒卡片时（**即流程的 Phase 1 开始前**）。
    
*   **作用：** 它是 App 的"遥控器信号线"。App 通过这条线给卡片下发指令，比如："去扫描耳机"、"连这个 MAC 地址"、"电量还有多少"。
    
*   **特点：** 它**不能**传输音频，只传极小的数据包，默认长连接且极度省电。
    

#### 第 2 次连接：手机系统 ↔ 卡片【下行音频链路】

*   **连接类型：** 经典蓝牙 (Classic BT - 支持 HFP/A2DP 协议)
    
*   **发生时机：** 用户跳出 App，跑到手机系统蓝牙设置里，手动点击连接 `Eureka-Audio` 时（**即步骤 2 到 3**）。
    
*   **作用：** 它是"音频输入线"。这一步是欺骗手机，让手机以为卡片是一个"外接蓝牙音箱/麦克风"。这样手机上的会议声音、通话声音才会流向卡片，卡片才能把声音截取下来录音。
    

#### 第 3 次连接：卡片 ↔ 蓝牙耳机【桥接音频链路】

*   **连接类型：** 经典蓝牙 (Classic BT)
    
*   **发生时机：** 用户在 App 里选定了自己的耳机，卡片主动去连接耳机时（**即步骤 6**）。
    
*   **作用：** 它是"耳机监听 + 麦克风采集"线路。卡片作为主机，连上耳机后承担双向音频桥接：
    *   **下行（对方声音 → 耳机）**：手机 → 卡片（第 2 条链路）进入卡片后，卡片将该音频**编码录音**，同时转发到耳机供用户监听（用户因此能听到对方）。
    *   **上行（我的声音 → 卡片录音）**：耳机麦克风采集用户语音 → 经第 3 条链路进入卡片 → 卡片将其**编码录音**；同时，卡片通过第 2 条链路将上行音频回传手机，以维持通话/会议链路畅通。
    *   **录音落点**：双向音频的**录音编码与文件存储均在卡片侧**完成。手机仅作为通话/会议的音频路由节点，不负责录音。
