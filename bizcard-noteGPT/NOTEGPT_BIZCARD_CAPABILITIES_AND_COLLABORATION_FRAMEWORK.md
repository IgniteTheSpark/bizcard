# NoteGPT 与 BizCard：能力对齐与合作逻辑

> 把双方能力说清楚，把这次联名硬件合作的逻辑讲明白。具体做成什么产品、什么功能，用单独的方案文档去落。

---

## 一、NoteGPT 的能力与特点

**抽象看**：各种内容当输入，AI 加工一遍，给出用户想要的结果。

- **输入**：很宽——录音、视频、PDF、PPT、网页、图片、文本都能进。
- **处理**：转写、总结、笔记、抽认卡、思维导图、问答、翻译等。
- **输出**：逐字稿、总结、笔记、抽认卡、问答、思维导图等，按需组合。

所以 NoteGPT 的一大特点是**输入大而广**，不同工具（Transcriber、Summarizer、Study、PDF、Chat 等）都是「不同输入 × 不同处理 × 不同输出」的组合。

| 维度 | 特点 |
|------|------|
| 形态 | 网页端、浏览器扩展为主，没有自己的硬件。 |
| 输入 | 录音、视频、PDF、网页、图片等都行，覆盖多场景。 |
| 短板 | 没有线下「一按即录」的入口，录音要用户先打开网页/插件，课堂、会议这种场景会断一截。 |

---

## 二、BizCard 的能力与特点

**抽象看**：硬件把**录音**这件事做实——线下一按就录，同步到 App；软件把录音变成**逐字稿**，还能让用户**标重点**。

- **硬件**：录音卡（可贴可立）、E-ink 屏、NFC、可做划重点/闪念等交互。
- **软件**：ASR、逐字稿（带时间戳）、用户标记重点；App 承载总结、待办等（当前偏会议/商务，可扩展）。

所以 BizCard 的特点是**以录音为单一、稳定的物理输入**，线下不用掏手机就能录，再交给后端处理。

| 维度 | 特点 |
|------|------|
| 形态 | 软硬一体，硬件是录音卡，App 配套。 |
| 输入 | 就是录音；一按即录，不依赖先打开网页或 App。 |
| 优势 | 线下录的体验好、有屏有交互，ASR 和逐字稿能输出成结构化数据给下游用。 |

## 三、这次合作的逻辑

### 3.1 输入从「大而广」变成「以录音为主」

- NoteGPT 自己玩的时候：输入什么都有，视频、PDF、网页、录音……
- 和 BizCard 硬件绑在一起之后：硬件能稳定提供的输入就是**录音**——课上、会上、访谈里一按就录，再进 App。
- 所以联名产品里，**对 AI 的输入会从「大而广」收成「以录音（和转出来的文本）为主」**；别的输入要是也有，也是 App 里可选的一块，主链路是录音。

### 3.2 主考虑的是「录音变成文本之后的事」

既然主输入是录音（和转录文本），这次合作主要想清楚的是：**录完、转成文字之后，做什么、怎么拓展。**

- 也就是：**录音 → 转写/逐字稿 → 后面怎么用？**  
  比如：出总结、出笔记、出重点、出抽认卡、出问答；和课程/会议挂上；和打卡、目标、排行这些玩法结合；硬件上再做个划重点、复习提醒之类的。
- **不需要**在联名产品里把 NoteGPT 那一整套「多输入入口」全搬过来；主链路就是「录音 → 文本 → 后面的场景和拓展」，多源能力可以当补充。

### 3.3 能力怎么切

| 谁 | 做什么 |
|----|--------|
| **BizCard** | 出硬件（录音卡、E-ink、划重点等）；出 ASR 和**清晰的逐字稿**；支持用户在稿子上**标记重点**；把 transcript（文本 + 时间戳 + 标记）交给下游。 |
| **NoteGPT** | 接 transcript，做**所有 AI 那一段**：总结、笔记、抽认卡、问答、思维导图等；在联名 App 里提供多少能力（只做录音这一条链，还是也塞一点视频/PDF 等），按具体方案来。 |

数据、账号、品牌、商业怎么算，合作里再对；这里只把「能力对齐」和「合作逻辑」说清。

## 四、小结

| 主题 | 一句话 |
|------|--------|
| **NoteGPT** | 输入大而广，AI 处理后多形态输出；网页端、无硬件，缺的是线下录音入口。 |
| **BizCard** | 以录音为入口，软硬一体，把「录 → 逐字稿 → 用户标重点」做实，能输出结构化 transcript。 |
| **合作逻辑** | 联名后主输入变成「录音（+ 转录文本）」；重点想的是**录音/文本进来之后**的场景和拓展；BizCard 管录和稿，NoteGPT 管 AI 加工和输出。 |
