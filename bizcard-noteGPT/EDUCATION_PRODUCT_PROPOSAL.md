# 教育产品方案：独立完整的学生向 App + 录音卡硬件

> **文档性质**：从 [BIZCARD_NOTEGPT_COLLABORATION_PROPOSAL.md](./BIZCARD_NOTEGPT_COLLABORATION_PROPOSAL.md) 中抽离的**教育向产品**专项方案，供产品与研发直接使用。  
> **背景**：BizCard 与 NoteGPT 在「录音卡片」上联合推出硬件，本方案描述该硬件背后的**学生向 App** 的完整形态，以及双方能力分工、合规与 NoteGPT 能力植入。  
> **状态**：方案稿 | **更新**：2026-02

---

## 一、产品定位

- **一款独立、完整、学生导向的 App**：从进入、使用到分享，心智统一为「学生的课堂与复习助手」；硬件是课堂入口，App 是主阵地。
- **联合产品** = 联合硬件（BizCard 录音卡，含划重点模式）+ 本 App（学生向功能与 NoteGPT 能力植入）。
- 教育线**不依赖名片、联系人**，专注课堂录音、笔记、复习、轻社交与排行榜。

---

## 二、能力分工（BizCard vs NoteGPT）

| 环节 | 负责方 | 具体能力 |
|------|--------|----------|
| 线下录音采集 | BizCard（硬件） | 课堂场景一按即录、同步上传。 |
| **ASR（语音转文字）** | **BizCard** | 做好 ASR，产出**清晰逐字稿（transcript）**。 |
| **逐字稿与标记** | **BizCard** | App 内呈现逐字稿，**允许用户标记重点**（高亮、打点、分段）；输出带时间戳 + 用户标记的结构化 transcript。 |
| **AI 总结、笔记、问答卡、题库、多源学习** | **NoteGPT** | 接收 transcript 做重点/笔记/问答卡/题库；并在 App 内**植入 NoteGPT 其他能力**（视频/PDF/网页总结、AI 问答等），供用户直接使用（见第五节）。 |

**接口约定**：BizCard 向 NoteGPT 输出 **transcript（文本 + 时间戳 + 用户标记的 highlight 区间）**；NoteGPT 负责所有 AI 侧处理与展示；账号与内容归属按双方商业协议约定。

---

## 三、Onboarding：学校、学级、课程表

- **首次进入**即收集学生向信息：
  - **学校**：用户选择或输入所在学校（支撑同校排行、同校笔记广场等）。
  - **学级**：年级/学段（如高一、大二），支撑同级排行与内容推荐。
  - **课程表**：onboard 时即可录入或导入本学期课表（课程名、时间、教室等）；也可后续补全，产品上引导「先建课表再用好 App」。
- 学校、学级、课程表由**用户主动填写或导入**，不自动从教务拉取，避免合规与接口依赖。

**课表与追踪**：

- 用户导入课表 → App 存为「课程 + 时间 + 可选教室/老师」。
- 用户某时段录音（或课后上传）→ 用户选择「归属哪门课」或 App 按**时间**推荐「是否关联到《高数》第 3 节」→ 确认后该条录音/笔记挂到该课程下。
- App 根据课表：展示「今日/本周有哪些课」、提醒（如「10 分钟后高数课」「高数课还没有本堂笔记」）、目标与 Streak（如「本周 5 节课已有 3 节有笔记」）、按课程维度的排行榜与统计。

---

## 四、内容流：课堂录音 → 笔记 / 问答卡 / 题库，挂载课程

| 环节 | 说明 |
|------|------|
| **录音** | 课堂用硬件录制，同步到 App。 |
| **自动总结成笔记** | 录音经 ASR → 逐字稿（含用户打点重点）→ AI 生成**结构化笔记**，自动挂到用户选择的**对应课程**下。 |
| **问答卡** | 由笔记/重点生成**问答卡**（抽认卡），用于复习；同样挂载到该课程。 |
| **相关题库** | 提供与本节课/本课程相关的**题库**（AI 根据笔记/重点生成或接入既有题库），练题与自测与课程绑定。 |
| **挂载到对应课程** | 每条录音、每条笔记、每套问答卡、每次练题记录，都**归属到用户课表中的某一门课**，形成「按课程维度的学习轨迹」。 |
| **分享与公开** | 仅限**用户自己整理后的笔记、重点、问答卡**等衍生内容，**不得分享录音本身或完整逐字稿**（合规见第八节）。用户可选择将某条/某节笔记、某套问答卡与**同课程小伙伴**共享（仅好友、同班、同校等可控档位）；产品需提示用户遵守校规与当地法律。 |

## 五、NoteGPT 能力植入 App：允许用户在 App 内使用

除「课堂录音 → 逐字稿 → 重点/笔记/问答卡」这条主链路外，**将 NoteGPT 的相应能力植入本 App**，允许用户在同一款 App 内直接使用，形成「课堂 + 多源自学」的一站式学生助手。

**可植入的 NoteGPT 能力示例**：

| 能力 | 说明 | 在 App 中的使用方式 |
|------|------|----------------------|
| **视频总结** | YT、B 站等视频一键总结、要点提取、带时间戳笔记 | 用户粘贴视频链接或上传本地视频，在 App 内生成总结/要点/思维导图；可选挂到某门课程下作为「拓展资料」。 |
| **PDF / 网页总结** | 上传 PDF 或输入网页链接，生成摘要、要点、问答 | 用户上传课件/论文/文章，在 App 内得到总结与问答，用于预习或复习；可关联到对应课程。 |
| **AI 问答** | 针对任意已上传/已总结内容提问，多模型（如 ChatGPT/Claude/DeepSeek） | 用户对某节课笔记、某段视频总结、某份 PDF 发起提问，在 App 内直接得到回答，无需跳转网页。 |
| **抽认卡 / 思维导图** | 由任意文本/总结生成抽认卡或思维导图 | 不仅课堂笔记可生成问答卡，用户从视频/PDF/网页得到的总结，也可在 App 内一键生成抽认卡或思维导图，统一纳入复习流。 |
| **多语言转录与翻译** | 40+ 语言转写与翻译 | 外语课、外文资料在 App 内转写与翻译，与笔记流统一。 |

**产品设计要点**：

- **入口统一**：上述能力在 App 内以「学习工具」或「多源输入」入口呈现（如「添加视频/PDF/网页」），与「课堂录音」并列，数据与账号统一，避免用户来回切换网页版 NoteGPT。
- **与课程可选关联**：用户可将某次视频总结、某份 PDF 总结**挂到某门课**下，与课堂笔记一起形成该课程的完整学习档案。
- **用量与计费**：植入能力的使用次数/时长可与 NoteGPT 现有订阅或联合产品的会员体系对接，具体按双方商业协议（如按次、包月、硬件用户专享额度等）。

**小结**：本产品中**同样可以把 NoteGPT 的相应能力植入 App**，使用户在 App 内即可使用视频总结、PDF/网页总结、AI 问答、抽认卡、思维导图等，而不必只依赖课堂录音；课堂录音是**主场景**，多源学习是**补充与增强**，共同构成完整的学生向体验。

## 六、硬件：划重点模式，课堂打点

- **硬件增加「划重点」模式**：在课堂录音过程中，用户可通过硬件**打点**（如单击/双击按键）在**当前时间点**标记「这里是重点」。
- 打点与录音一起上传，在逐字稿中体现为**高亮区间或重点标记**，供 AI 生成笔记时优先关注，也便于用户回看时快速定位。
- **课堂上无需掏手机**即可完成「录 + 划重点」。

硬件还可参与**复习/目标展示**：如 E-ink 显示「今日 Streak」「本周排名变化」或「还差 1 次复习达成今日目标」，把 engagement 延伸到线下。

---

## 七、Engagement 与排行榜（多邻国式 + 课表维度）

- **Engagement**：连续学习天数（Streak）、每日/每周目标（如「本周 3 节课有笔记」「完成 5 次抽认卡」）、XP/等级、碎片化复习任务（硬件「一键复习」、App「今日一题」等）。
- **排行榜**：**轻量**，维度与隐私可控：
  - **同学校**：同校用户的学习时长、笔记数、复习次数、连续天数等排名。
  - **同学级**：同年级/学段排名。
  - **同地区**：同城市/省份（可选）。
- 支持「仅好友」「同班」「匿名」等档位；强调「进步」与多元维度，避免唯分数与恶性攀比。

---

## 八、课堂录音的法律与合规：不能用作公开或分享

**课堂录音是否违法？** 取决于法域、学校规定和具体用途；**将课堂录音（或完整逐字稿）用于公开或分享，法律与校规风险明显更高。**

| 风险 | 产品策略 |
|------|----------|
| **录音本身** | 不提供对**录音文件**的对外分享或公开；仅限用户本人访问与使用。 |
| **完整逐字稿** | 不提供对**完整逐字稿**的分享或公开；仅限用户本人使用。 |
| **可分享内容** | 仅允许分享**用户自己整理、加工后的内容**（结构化笔记、重点摘要、问答卡），标注为「个人学习笔记」；分享范围可控，并在分享前**提示用户**遵守校规与当地法律。 |
| **产品提示** | App 内通过使用条款、首次录音/首次分享前的合规提示，说明：课堂录音需遵守校规与法律；禁止将录音或逐字稿对外传播；分享仅限本人衍生内容且用户需对合规负责。 |

## 九、文档小结

| 维度 | 要点 |
|------|------|
| **定位** | 独立、完整的学生向 App + 录音卡硬件；课堂与复习助手，不主打名片。 |
| **分工** | BizCard：硬件、ASR、逐字稿、用户标记重点；NoteGPT：所有 AI 能力（重点/笔记/问答卡/题库/多源总结/问答等），并**在 App 内植入多源能力**供用户直接使用。 |
| **Onboarding** | 学校、学级、课程表；课表用于追踪课堂与记录、目标与排行。 |
| **内容流** | 录音→笔记/问答卡/题库，全部挂载到课程；分享仅限衍生内容，不含录音/逐字稿。 |
| **NoteGPT 植入** | 视频/PDF/网页总结、AI 问答、抽认卡、思维导图、多语言等能力在 App 内开放，形成课堂 + 多源自学一站式体验。 |
| **硬件** | 划重点模式（课堂打点）；可选 Streak/目标展示。 |
| **Engagement** | Streak、目标、XP、轻量排行榜（同校/同级/同地区）。 |
| **合规** | 录音与逐字稿仅限个人使用，不公开不分享；分享仅限用户笔记/问答卡并提示合规。 |
