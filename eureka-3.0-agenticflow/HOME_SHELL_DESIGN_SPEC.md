---

## 1. 命题

**首页主视觉**是一条 **时间轴（Timeline）**，向用户反馈与其直接相关的内容与事件流（与 [工作台时间轴](./WORKBENCH_TIMELINE_DESIGN_SPEC.md) 的「来源 / 产物」结构一致或为其超集）。

与此同时，用户还需要 **不依赖时间轴扫读** 也能稳定到达的 **全局导航** 与 **快速生产入口**。本文描述这些模块在首页上的 **角色分工与层级关系**，**不规定** 顶栏/底栏/悬浮的具体像素与组件样式。

---

## 2. 信息层级（建议）

| 层级 | 模块 | 作用 |
|---:|---|---|
| **L0 主区** | **Timeline** | 主滚动区；用户的「直接相关内容」首选呈现位置。 |
| **L1 全局列表入口** | **Files**、**Workspace**、**Calendar** | 按维度汇总浏览：文件库、结构化资产、时间与日程视图；与时间轴 **互补**（列表管理 vs 时间流叙事）。 |
| **L2 账户与设备** | **个人中心**、**设备连接** | 账号/设置/同步等；硬件连接状态与管理入口。 |
| **L3 快捷生产** | **Capture**、**Ask Agent**、**上传（Upload）** | 高频动作持续可达；触发采集、对话、显式上传，产出 **回流到时间轴** 的新行或新里程碑。 |

---

## 3. 各入口的职责边界（逻辑）

### 3.1 Files（文件列表）

- 提供 **全库视角**：按类型、文件夹、状态等浏览 File。
- 与时间轴关系：时间轴上的「文件」行是 **事件切片**；Files 列表是 **库存与管理视图**。二者应可交叉跳转（从轴进详情 ↔ 从列表进同一文件）。

### 3.2 Workspace（资产列表）

- 提供 **结构化产出** 的汇总：Notes / Reminders(Todos) / Ideas / Contacts 等（具体集合与 `DEMO_SPEC` / 架构文档一致）。
- 与时间轴关系：时间轴右侧 **产物列** 与 Workspace **同源不同形**——轴上强调「哪次来源产生了什么」，Workspace 强调「按类型管理与检索」。

### 3.3 Calendar（日历）

- 提供 **时间维度** 的日程与提醒视图（若团队已将 Todo 主入口放在 Calendar，保持与 [HOME_LOGIC_SPEC](./HOME_LOGIC_SPEC.md) 一致即可）。
- 与时间轴关系：Calendar 偏 **计划与到期**；时间轴偏 **实际发生的事与 AI 产出**。二者可在单条 Reminder / 事件上 deep link，本文不展开字段。

### 3.4 个人中心

- 账户、偏好、关于、同步与隐私等 **与用户身份相关** 的配置与信息。
- **不作为** 主内容流的一部分出现在时间轴内，避免与「工作事件」混淆。

### 3.5 设备连接

- 硬件被发现、配对、连接状态、副屏/录音设备等 **设备域** 入口。
- 与时间轴关系：设备动作（如录音开始）可能 **触发时间轴新行**；入口本身保持独立导航语义。

### 3.6 快捷：Capture

- **快速采集**：短录音、闪念硬件路径等（具体能力与硬件 PRD 对齐）。
- 预期：**产生新的来源事件**，在时间轴上出现对应 **来源行**（或后续里程碑）。

### 3.7 快捷：Ask Agent

- **快速开启对话**（conversation Session），不替代时间轴内对历史会话行的展示。
- 预期：新开会话后，用户可在时间轴规则下看到 **会话级条目**（与 [FEED_MODE_DEMO_SPEC](./FEED_MODE_DEMO_SPEC.md) 中 Ask 叙事一致时可对齐）。

### 3.8 快捷：上传（Upload）

- **显式上传** 文件类输入（录音 / 图片 / Markdown 等），与系统分享入口互补。
- 预期：**文件入库里程碑** 出现在时间轴；后续解析里程碑 **另起行或同文件线程**，由事件模型决定。

---

## 4. 布局与注意力（原则性，不锁样式）

1. **Timeline 占据主阅读区**：导航与快捷入口 **不应长期遮挡** 时间轴核心阅读区域；可采用顶栏、底栏、侧缘或收起态悬浮等方案，由视觉稿落地。
2. **L1 三项（Files / Workspace / Calendar）** 应在任意时刻 **可被发现**（一眼可知去哪找列表），避免完全藏进二级菜单而无替代路径。
3. **L3 快捷三项** 强调 **持续可达**（固定悬浮条、浮动胶囊、底栏扩展等），与时间轴内容区分 **可操作性与内容层**（例如避免与右侧 Asset 条误触混淆——具体间距与热区由 UI 实现）。
4. **安全区**：快捷入口需避开系统手势区域；具体 inset 随设备规范。
