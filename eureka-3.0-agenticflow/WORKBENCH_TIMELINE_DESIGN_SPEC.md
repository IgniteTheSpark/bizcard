## 1. 目的与适用范围

### 1.1 目的

定义 **AI 工作台时间轴** 在界面上的 **结构与交互逻辑**：如何将「来源主体」与「衍生 Asset」组织在同一行叙事内，并保证可读性、可扩展性与多端一致性。

本文 **不规定** 配色、圆角、字号等视觉细则；实现时可参考 `demo/workbench-timeline-demo.html` 作为布局原型。

### 1.2 适用范围

- 移动端工作台首页或等价「事件时间轴」视图。
- 与底层对象模型（File / Session / Asset）对齐；不改变领域模型，仅定义 **呈现与导航规则**。

### 1.3 非目标

- 不定义后台事件写入顺序或服务端 Schema。
- 不收敛「状态机」枚举（待解析 / 同步中等具体文案可由业务迭代）。
- 不替代 Session 工作台、文件详情页的完整功能定义。

---

## 2. 核心隐喻：一行一事，左右分流

每一 **时间轴行（row）** 表达 **一次可追溯的来源事件** 及其在当前产品上可见的 **产出集合**。

| 区域 | 语义 | 用户心智 |
|---|---|---|
| **左侧** | **来源（subject）** | 「这件事从哪来的」——文件入库、闪念输入、Ask 会话、手动创建等。 |
| **中间** | **时间轴轴线（axis）** | 「时间上的一颗节点」，连接前后事件；同时承担 **左与右的视觉分隔**。 |
| **右侧** | **产物（outputs）** | 「由此衍生出的 Asset」——Note / Todo / Idea / Contact 等；可为空。 |

**结论**：同一行的左侧与右侧是 **1 : N** 关系（N≥0）。左侧一条来源卡片最多对应右侧多条 Asset 条目；若 N=0，右侧 **不出现占位文案**，仅保留左侧与轴线（产品可选择是否弱化该行视觉权重，本文不规定）。

---

## 3. 布局逻辑（不绑定像素）

### 3.1 三栏结构

- **来源区**、**轴线区**、**产物区** 并列。
- **产物区水平占比应大于来源区**，避免 Asset 列表可读性被挤压；具体比例由设计实现，不作为 Spec 硬编码。
- **轴线位于来源与产物之间**，作为主轴分隔，而非贴在最左边缘。

### 3.2 垂直对齐

- 单行高度由 **本行三块内容的天然高度** 决定：**左侧来源卡片**、**轴线拉伸高度**、**右侧产物栈** 三者 **顶对齐或沿主轴等高拉伸**（实现任选其一，原则是轴线贯穿该行可视高度）。
- 当右侧 **无产物** 时，行高主要由 **左侧来源块** 决定；轴线仍与本行等高。

---

## 4. 来源侧（左侧）：统一容器逻辑

### 4.1 统一容器

- 每一种来源类型（文件 / 闪念 / Ask / 手动）均在 **同一套容器语义** 内展示：**标签区分类型**，下方承载标题或摘要，整块可点击。
- **不要求** 各类型像素级同款样式，但要求 **信息层级一致**：类型可识别 → 主文案 → 次要说明（若有）。

### 4.2 文件类来源

展示信息至少包含：

- **类型维度**：文件实体 + 介质类型（音频 / 图片 / Markdown 等）。
- **状态维度**：与业务一致的生命周期状态（如待解析、待同步、已解析等）；枚举由业务定义，本文只要求 **状态可被扫读且与详情一致**。
- **标题规则**：
  - 未完成解析或尚无正式标题时：使用占位文案（如新录音、新图片）。
  - Markdown 等可直接取文件标题。
  - 解析完成后可被 **规范化标题** 替换（与文件详情一致）。

点击行为：**进入该文件详情**。

### 4.3 闪念类来源

- 技术上归属某 **flash_note Session**；在来源区 **突出「闪念」标识**，与普通 Session 区分。
- 展示 **ASR 或等效文本摘要**，过长时 **截断 + 展开**，展开为客户端交互，不新增时间轴行。

点击行为：**进入闪念 Session，并定位到对应 input / turn**（锚点规则由 Session 页实现）。

### 4.4 Ask Agent 类来源

- 展示 **用户本轮输入摘要**（同上：截断 + 展开）。
- 与普通闪念区分标识（如 Ask Agent / conversation）。

点击行为：**进入 conversation Session，并定位到对应 input / turn**（与 flash 分流）。

### 4.5 手动类来源

- 标识 **手动** + **创建的 Asset 类型**（新建 Todo / Note / Contact 等）。
- 主文案为标题或简介。

点击行为：**进入对应 Asset 详情**（或等价编辑上下文）。

### 4.6 左侧等高（逻辑要求）

- **同一列表内**，来源容器在 **未展开摘要全文** 的前提下，采用 **统一可视高度策略**，避免相邻行跳跃造成扫读困难。
- 展开摘要导致高度变化 **仅影响当前行**，符合预期。

---

## 5. 产物侧（右侧）：1 : N 与默认展示条数

### 5.1 条目语义

- 右侧每一条对应 **一个可跳转的 Asset**（或产品定义的等价最小产出单元）。
- 每条 **等高** 指 **同一列表内条目采用统一行高策略**，便于扫读；具体数值由 UI 实现。

### 5.2 默认条数与展开

- 同一来源 **默认最多展示 K 条**（产品常取 **K=3**）。
- 若 N>K：**隐藏多余条目**，并提供 **「还有 (N−K) 条 · 展开」**（文案可调）。
- **展开** 在本行内露出全部条目，**纵向拉长该行**；时间轴总长度随之增加。
- **收起** 是否提供：可选；若提供，不应丢失已读状态以外的业务含义。

### 5.3 无产物

- N=0：**右侧不渲染占位符**（不出现「无产出」等文案），由布局自然留出空白或与左卡底部对齐，具体对齐方式由实现选择。

---

## 6. 轴线（中间）：分隔与节点

### 6.1 作用

- **分隔** 来源与产物两列。
- **标记** 该行在时间序列上的节点（与前后行轴线连接，形成连续时间隐喻）。

### 6.2 样式自由度

- 竖线连续方式、节点形状、与文本的对齐，由视觉设计决定；本文只要求 **语义位置在中间列、贯穿该行高度**。

---

## 7. 导航与点击分层

| 操作对象 | 预期行为 |
|---|---|
| 左侧来源容器（整体） | 按来源类型跳转：文件详情 / Session+锚点 / Asset 详情等。 |
| 右侧单条 Asset | 进入该 Asset 详情。 |
| 摘要「展开/收起」 | 仅影响文案展开，**不触发** 来源导航；事件需与容器点击解耦。 |
| 「还有 N 条 · 展开」 | 仅展开列表，**不触发** 来源导航（除非产品刻意合并，默认不合并）。 |

---

## 8. 与里程碑数据的关系

- 时间轴 **一行** 对应产品上的 **一次「来源级事件」**（例如文件在某时刻完成解析并产出一批 Asset）。
- 底层可映射为 `FEED_MODE_DEMO_SPEC` 中的 `milestone` + `refs`；本文定义的 UI 是其在 **工作台视图** 上的一种呈现策略：**左侧锚定来源，右侧列出 refs 中的 Asset 子集**。
